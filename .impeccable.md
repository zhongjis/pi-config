## Design Context

### Users
Primary user is the project owner: a dense power-user terminal workflow user working inside pi for long coding sessions. The interface should support fast repeated glances without requiring careful reading.

### Brand Personality
Quiet, minimal, refined. Calm confidence over flashy decoration. Information-dense, but never visually loud.

### Aesthetic Direction
Terminal-native and editorial rather than dashboard-like. Prefer restrained contrast, concise separators, and strong hierarchy through placement and emphasis instead of extra ornament. Optimize for glanceability first and density second.

### Design Principles
1. Put the most actionable signal first.
2. Remove static or redundant labels before removing useful data.
3. Use color sparingly, only for warning/error emphasis.
4. Prefer concise grouping and alignment over decorative separators.
5. Keep secondary state available, but visually subordinate to core session info.
